How To Deploy on Alibaba Cloud

Create the ECS Instance
Via the Console
1. Log in to the ECS Console and click Create Instance.
2. Select Custom Launch for full control.
3. Choose a billing method
4. Select Region & Zone.
5. Pick your Instance Type from the families listed above.
6. Choose an Image — Ubuntu 22.04 or 24.04 LTS is recommended for AI workloads.
7. Configure Storage
8. Set up Networking.
9. Attach a Security Group.
10. Set Login Credentials — use a Key Pair (recommended over passwords).
11. Under Advanced, set a descriptive instance name, optionally attach a RAM role, and
add User Data scripts for automated bootstrapping.
12. Review, accept the Terms of Service, and click Create Order.

Set Up Networking
VPC: Create a VPC with a private CIDR block (e.g., 172.16.0.0/12). Create VSwitches across
multiple zones if you plan to use Auto Scaling or load balancing later.
Public IP: Either assign a public IPv4 address directly (pay-by-bandwidth or pay-by-traffic) or
attach an Elastic IP (EIP) for flexible association. For backend instances that don't serve internet
traffic directly, skip the public IP and use a NAT Gateway for outbound access.
Security Groups: These act as whitelists (deny-all by default). Open only the ports your agent
needs:
Port Protocol Source Purpose
22 TCP Your IP only SSH access
80 TCP 0.0.0.0/0 HTTP (if serving web

traffic)
443 TCP 0.0.0.0/0 HTTPS
8080 TCP Your IP or LB Agent API (if applicable)

Never open SSH (22) or RDP to 0.0.0.0/0 in production.

Connect to the Instance
Method Best for
Workbench (browser-based) Quick access, no local software needed
SSH (OpenSSH / PuTTY) Standard terminal access
VNC Console Troubleshooting when SSH fails
Session Manager Auditable access without a public IP

Deploying an Agent on Alibaba Cloud SAS

When to Use SAS Instead of ECS
Simple Application Server (SAS/SWAS) bundles compute, storage, and networking into
predictable monthly plans. It trades flexibility for simplicity.

Choose SAS if... Choose ECS instead if...
You need a single agent server for dev/test or
low-traffic production

You need GPU instances for local model inference

You want fixed, predictable pricing with no
surprise bills

You need Auto Scaling or load balancing
You want to deploy in under 5 minutes You need full VPC control or VPC peering
Your agent calls external LLM APIs (no local GPU
needed)

Your workload is high-concurrency or
mission-critical

You're an individual developer, student, or small
team

You need multiple data disks or specialized
storage

Best-fit agent types for SAS: LLM API wrapper agents, Dify / MaxKB deployments, lightweight
RAG pipelines, chatbot backends, webhook-based automation agents.

Create the Instance
1. Go to the SAS Console.
2. Click Create Server.
3. Select a Region.
4. Select an Image.
5. Select your Plan
6. Choose quantity and subscription duration.
7. Complete payment. The instance is provisioned immediately with an auto-assigned
public IP.

Important: SAS instances have no default password. Before connecting, reset the root
password from the console under Reset Password.

Choose an Image

Configure the Firewall
SAS has a built-in firewall that controls inbound traffic only (outbound is unrestricted). It supports
up to 50 rules per instance.
Default rules (Linux): TCP 22, 80, 443 + ICMP.
Open the ports your agent needs:
Port Protocol Source Purpose
22 TCP Your IP range only SSH (restrict source)
80 TCP 0.0.0.0/0 HTTP
443 TCP 0.0.0.0/0 HTTPS
8080 TCP 0.0.0.0/0 or restricted Agent API endpoint
To add a rule: SAS Console > your instance > Firewall tab > Add Rule > select a preset (HTTP,
SSH, MySQL, etc.) or define a custom port range.
Security tip: Restrict SSH (port 22) to your known IP ranges rather than leaving it open to all
sources.
